# Mode Package Guidance

Mode packages are schema-supported but not published during Phase 18B.

A future marketplace-ready mode should define:

- a clear operator purpose
- prompt and behavior boundaries
- expected model and tool capabilities
- activation assumptions
- safety and permission posture
- compatibility range
- concise examples and validation expectations

Modes should not be added until NeonConductor first alpha is finished and the
mode import path is stable enough to validate real user-facing behavior.
